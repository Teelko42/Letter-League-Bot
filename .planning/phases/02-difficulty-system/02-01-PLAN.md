**src/difficulty/frequency.py** -- Implement:

```python
from __future__ import annotations

from wordfreq import get_frequency_dict


class FrequencyIndex:
    """Wraps wordfreq for O(1) per-word Zipf frequency lookups.

    Loads the full English frequency dictionary once at initialization.
    Per-word lookup is a simple dict .get() with a default for OOV words.

    The Zipf scale ranges from 0 (OOV/never seen) to ~8 (most common words like 'the').
    """

    OOV_ZIPF: float = 0.0
    MAX_ZIPF: float = 8.0  # approximate upper bound for normalization

    def __init__(self, lang: str = 'en') -> None:
        """Load the full frequency dictionary for the given language.

        This is a one-time cost (~1-2s, ~10MB RAM). Instantiate once at
        bot startup, not per request.

        Args:
            lang: Language code for wordfreq. Default 'en' (English).
        """
        # get_frequency_dict returns {word: frequency_float}
        # The values are Zipf-scale frequencies (log10-based, 0-8 range)
        self._freq: dict[str, float] = get_frequency_dict(lang, wordlist='best')

    def zipf(self, word: str) -> float:
        """Return Zipf frequency for a word (0.0-8.0). OOV returns 0.0.

        Args:
            word: The word to look up (case-insensitive).

        Returns:
            Zipf frequency: ~8 for most common, ~0 for OOV/very rare.
        """
        return self._freq.get(word.lower(), self.OOV_ZIPF)

    def normalized(self, word: str) -> float:
        """Return frequency normalized to [0.0, 1.0]. 1.0 = most common.

        Normalizes against MAX_ZIPF (fixed constant ~8.0), not against
        the current word set. This prevents "most common among rare" from
        being treated as common.

        Args:
            word: The word to look up (case-insensitive).

        Returns:
            Normalized frequency in [0.0, 1.0].
        """
        return self.zipf(word) / self.MAX_ZIPF
```

**IMPORTANT:** The `get_frequency_dict` function returns a dict where values are Zipf-scale frequencies. Verify this by checking that `freq_dict.get('the', 0.0)` returns a value around 7-8. If `get_frequency_dict` returns raw frequencies instead of Zipf values, use `wordfreq.zipf_frequency()` to convert, or use `word_frequency` and convert manually. The research recommends the Zipf scale -- confirm the API returns it.

**Note on `get_frequency_dict` return format:** The wordfreq docs say `get_frequency_dict(lang)` returns `{word: frequency}` where frequency is a small float (NOT Zipf scale). If this is the case, use `wordfreq.zipf_frequency(word, lang)` for individual lookups instead, OR pre-compute Zipf values from the raw frequencies. The simplest correct approach: use `zipf_frequency(word, 'en')` inside the `zipf()` method, but cache results in a local dict on first access to maintain O(1) amortized. OR, pre-build the Zipf dict at init time:

```python
from wordfreq import zipf_frequency, get_frequency_dict

class FrequencyIndex:
    def __init__(self, lang: str = 'en') -> None:
        # Pre-load all known words with their Zipf frequencies
        raw = get_frequency_dict(lang, wordlist='best')
        self._freq: dict[str, float] = {
            word: zipf_frequency(word, lang)
            for word in raw
        }
```

However, this may be slow for large dictionaries. The more pragmatic approach:

```python
from wordfreq import zipf_frequency

class FrequencyIndex:
    def __init__(self, lang: str = 'en') -> None:
        self._lang = lang
        self._cache: dict[str, float] = {}

    def zipf(self, word: str) -> float:
        key = word.lower()
        if key not in self._cache:
            self._cache[key] = zipf_frequency(key, self._lang)
        return self._cache[key]
```

**Choose the approach that gives correct Zipf-scale values with the best performance for batch move-list processing.** The key requirement: `zipf('the')` should return approximately 7-8, `zipf('cat')` ~4-5, and OOV words return 0.0. Test this during RED phase to validate the API.
