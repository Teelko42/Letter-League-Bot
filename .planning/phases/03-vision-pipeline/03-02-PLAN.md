Create `src/vision/extractor.py` with one public async function:

```python
async def call_vision_api(img_bytes: bytes, retry_context: str | None = None) -> dict:
```

Implementation (follows 03-RESEARCH.md Pattern 1):

1. **Client:** Create module-level `AsyncAnthropic()` client (no API key argument — reads from `ANTHROPIC_API_KEY` env var by default).

2. **Base64 encode:** `base64.standard_b64encode(img_bytes).decode("utf-8")`

3. **Prompt construction:**
   - Define `EXTRACTION_PROMPT` as a module-level constant string. The prompt must:
     - State this is a Letter League game board screenshot
     - State the board is 19 rows x 27 columns (0-indexed, row 0 = top, col 0 = left)
     - State columns increase left-to-right, rows increase top-to-bottom
     - Instruct to count grid lines/borders for accurate coordinate mapping
     - Instruct to return ONLY cells with placed tiles (not empty cells)
     - Instruct to extract the player's tile rack (the row of tiles below the board)
     - Instruct to identify blank tiles by their visual appearance (lighter color/different border) and set is_blank=true
     - State each cell needs: row, col, letter (uppercase A-Z), is_blank, multiplier (the square's multiplier type)
     - State rack should be uppercase letters, including '?' for blank tiles in rack
   - If `retry_context` is provided, append it after the base prompt: `f"\n\nPREVIOUS ATTEMPT HAD ERRORS:\n{retry_context}\nPlease correct these issues."`

4. **API call:** `await client.messages.create(...)` with:
   - `model="claude-sonnet-4-6"` (per STATE.md decision — ~$0.004/screenshot)
   - `max_tokens=4096` (sufficient for most boards; research notes to increase to 8192 if truncation observed)
   - `messages` with image (type: "image", source: base64/png) followed by text prompt
   - `output_config={"format": {"type": "json_schema", "schema": BOARD_SCHEMA}}`
   - Import `BOARD_SCHEMA` from `src.vision.schema`

5. **Parse:** `json.loads(response.content[0].text)` — guaranteed valid JSON by structured outputs

6. **Error handling:** Wrap the API call in try/except. On `anthropic.APIError` or other exceptions, raise `VisNError(EXTRACTION_FAILED, str(e))`.

7. **Logging:** Use `loguru.logger` to log:
   - API call start (with retry_context presence)
   - Response usage tokens: `response.usage.input_tokens`, `response.usage.output_tokens`
   - Total latency (use `time.monotonic()` around the API call)

Imports: `json`, `base64`, `time`, `anthropic.AsyncAnthropic`, `loguru.logger`, schema and error modules.

Use `from __future__ import annotations`.

  <name>Task 2: Implement four-check validator with unit tests</name>
  <files>src/vision/validator.py, tests/vision/test_validator.py</files>
  <action>
**src/vision/validator.py** — One public function:

```python
def validate_extraction(data: dict) -> list[str]:
```

Returns a list of validation error messages (empty list = all checks pass).

Implements the four checks from CONTEXT.md (in order):

**Check 1 — Valid letters (A-Z only):**
For each cell in `data["board"]["cells"]`, verify `cell["letter"]` is in `ALPHABET` (import from `src.engine.tiles`). Error message: `f"Invalid letter '{cell['letter']}' at ({cell['row']}, {cell['col']})"`.

**Check 2 — Connectivity (no floating tiles):**
Use flood-fill from the first placed tile. All placed tiles must be reachable via orthogonal moves through other placed tiles. Steps:
- Build `placed: set[tuple[int, int]]` from all cell (row, col) positions
- If `len(placed) <= 1`: skip (single tile or empty board is trivially connected)
- Start BFS/DFS from any tile in `placed`
- Mark visited tiles, walk orthogonal neighbors `(r-1,c), (r+1,c), (r,c-1), (r,c+1)` that are also in `placed`
- After traversal: any tile in `placed` not visited is floating
- Error message: `f"Floating tile '{cell['letter']}' at ({r}, {c}) — not connected to other tiles"`

**IMPORTANT:** Use flood-fill (not per-tile neighbor check) per Pitfall 5 in research — avoids false positives on word endpoints.

**Check 3 — Multiplier positions match official layout:**
For each cell in `data["board"]["cells"]`, look up `(cell["row"], cell["col"])` in `OFFICIAL_MULTIPLIER_LAYOUT` (import from `src.vision.schema`). Expected value is the dict value, or `"NONE"` if position not in dict. Compare with `cell["multiplier"]`. Error message: `f"Multiplier mismatch at ({row}, {col}): got {cell['multiplier']}, expected {expected}"`.

**Check 4 — Rack count <= 7:**
If `len(data["rack"]) > 7`: error `f"Rack has {len(data['rack'])} tiles (max 7)"`.
Also validate each rack tile is in `ALPHABET + '?'` (? for blanks in rack). Error: `f"Invalid rack tile '{tile}'"`.

---

2. **test_invalid_letter_detected** — Cell with `letter: "3"` or `letter: "!"`. Assert error list contains "Invalid letter".

8. **test_invalid_rack_tile_detected** — Rack containing "1". Assert error mentions "Invalid rack tile".

  <name>Task 3: Wire pipeline entry point — extract_board_state public API</name>
  <files>src/vision/__init__.py</files>
  <action>
Update `src/vision/__init__.py` to be the public API module. Implement:

```python
async def extract_board_state(
    img_bytes: bytes,
    mode: str = "classic",
) -> tuple[Board, list[str]]:
```

Pipeline steps (sequential):

1. **Preprocess:** Call `preprocess_screenshot(img_bytes)` → `processed_bytes`. If it raises `VisNError`, let it propagate (INVALID_SCREENSHOT for bad images).

2. **Extract:** Call `await call_vision_api(processed_bytes)` → `data` dict. If it raises `VisNError`, let it propagate (EXTRACTION_FAILED).

3. **Validate:** Call `validate_extraction(data)` → `errors: list[str]`.

4. **Retry on failure:** If `errors` is non-empty:
   - Build retry context: `"\n".join(errors)`
   - Log warning with `loguru.logger`: `f"Validation failed ({len(errors)} errors), retrying: {errors}"`
   - Call `await call_vision_api(processed_bytes, retry_context=retry_context)` → `data` (second attempt)
   - Call `validate_extraction(data)` → `errors` again
   - If still non-empty: raise `VisNError(VALIDATION_FAILED, f"Validation failed after retry: {'; '.join(errors)}")`

5. **Populate Board:** Create `Board(rows=19, cols=27, multiplier_layout=multiplier_layout_engine_format)` where `multiplier_layout_engine_format` is built from `OFFICIAL_MULTIPLIER_LAYOUT` + `MULT_STR_TO_ENGINE`:
   ```python
   multiplier_layout_engine = {
       pos: MULT_STR_TO_ENGINE[mult_str]
       for pos, mult_str in OFFICIAL_MULTIPLIER_LAYOUT.items()
   }
   ```
   (This can be computed once at module level as a constant.)

   For each cell in `data["board"]["cells"]`:
   ```python
   board.place_tile(
       row=cell["row"],
       col=cell["col"],
       letter=cell["letter"],
       is_blank=cell["is_blank"],
       mode=mode,
   )
   ```

6. **Extract rack:** `rack = data["rack"]` — list of uppercase letter strings.

7. **Return:** `(board, rack)`
