```python
MAX_WORD_RETRIES = 3        # max different words to try before tile swap
RECALL_X_FRAC = 0.13        # PLACEHOLDER — recall/undo button X (fraction of canvas width)

```

**Add to CoordMapper:**
- `recall_btn_px(self) -> tuple[float, float]` — returns viewport pixel coordinates for the recall/undo button (clear placed tiles before re-trying).
- `swap_btn_px(self) -> tuple[float, float]` — returns viewport pixel coordinates for the tile swap button.

**Add to TilePlacer:**

- `async _click_confirm(self, mapper: CoordMapper) -> None` — click the confirm button with jitter. Uses `page.mouse.click(x, y)` (teleport, no drag per CONTEXT.md). Log the click coordinates.

- `async _wait_for_acceptance(self) -> bool` — wait `random.uniform(1.0, 2.0)` seconds (per CONTEXT.md), then capture a screenshot and call `classify_frame()`. Return `True` if state is NOT `"my_turn"` (meaning turn ended = word accepted). Return `False` if still `"my_turn"` (meaning word was rejected). Import `classify_frame` from `src.browser.turn_detector`.

- `async _recall_tiles(self, mapper: CoordMapper) -> None` — click the recall/undo button to clear placed tiles from the board back to rack. Click with jitter, then wait `asyncio.sleep(random.uniform(0.5, 1.0))` for the animation to complete. Log the action.

- `async _tile_swap(self, mapper: CoordMapper) -> None` — click the tile swap button to perform a tile swap action when no valid words can be placed. Click with jitter. Log a warning that tile swap was used as a fallback.

- `async place_move(self, moves: list[Move], rack: list[str]) -> bool` — the public API that orchestrates the full placement + confirmation flow:
  1. Iterate through `moves` (up to `MAX_WORD_RETRIES` attempts).
  2. For each move:
     a. Call `self.place_tiles(move, rack)` to drag all tiles.
     b. Get a fresh bbox and create a CoordMapper.
     c. Call `self._click_confirm(mapper)`.
     d. Call `self._wait_for_acceptance()`.
     e. If accepted (True): log success (word, score), return `True`.
     f. If rejected (False): log rejection, call `self._recall_tiles(mapper)`, continue to next move.
  3. If all word attempts exhausted:
     a. Get a fresh bbox, create a CoordMapper.
     b. Call `self._tile_swap(mapper)`.
     c. Log that tile swap fallback was used.
     d. Return `False`.
  4. Wrap the entire method in a try/except for `PlacementError` — log the error, attempt recall, and continue to next move. If the last move also fails with PlacementError, fall through to tile swap.

Use loguru logger throughout. Follow the project convention of `from __future__ import annotations` and TYPE_CHECKING guard for type imports.

**Important:** The `place_move()` method receives a `list[Move]` sorted by score descending (best first). This is the list from `find_all_moves()` or a subset. Plan 02 does NOT call the engine — the caller provides the move list. The retry loop simply tries the next move in the list.
