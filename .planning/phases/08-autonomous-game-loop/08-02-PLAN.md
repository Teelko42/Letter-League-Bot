Create `src/bot/autoplay_cog.py` implementing the full autonomous game loop cog. Follow the patterns from RESEARCH.md and the existing AdvisorCog closely.

**Cog structure:**

```python
class AutoPlayCog(commands.Cog):
    autoplay_group = app_commands.Group(
        name="autoplay",
        description="Autonomous game loop controls",
    )
```

**Instance attributes:**
- `self.bot: LetterLeagueBot`
- `self._loop_task: asyncio.Task | None = None`
- `self._stop_event: asyncio.Event = asyncio.Event()`
- `self._state: LoopState | None = None`
- `self._session: BrowserSession | None = None` (for crash recovery)

**Slash commands (3 subcommands):**

1. `@autoplay_group.command(name="start")` — `autoplay_start(self, interaction)`
   - Defer immediately (non-ephemeral so channel sees it)
   - Guard: if `self._state is not None and self._state.phase != AutoPlayPhase.IDLE`, send error "Autoplay is already running"
   - Read `VOICE_CHANNEL_URL` from `os.environ`; error if missing
   - Set `self._state = LoopState(phase=AutoPlayPhase.STARTING, channel_id=interaction.channel_id, channel_url=channel_url)`
   - Clear stop event
   - Create task: `self._loop_task = asyncio.create_task(self._run_game_loop(interaction.channel, channel_url), name="autoplay-game-loop")`
   - Send followup: `build_info_embed("Autoplay Started", "Launching browser and connecting...")`

2. `@autoplay_group.command(name="stop")` — `autoplay_stop(self, interaction)`
   - Guard: if state is None or phase is IDLE, send ephemeral "Not currently active"
   - Set `self._stop_event.set()` and `self._state.phase = AutoPlayPhase.STOPPING`
   - Send ephemeral message: "Finishing current turn, then stopping..."

3. `@autoplay_group.command(name="status")` — `autoplay_status(self, interaction)`
   - If state is None or phase is IDLE: send ephemeral "Autoplay is not running"
   - Otherwise: compute uptime from `time.monotonic() - self._state.start_time`, format as Xm Ys
   - Send ephemeral embed with fields: Status (state.phase.value), Turns (state.turn_count), Uptime

**Game loop coroutine `_run_game_loop(self, channel, channel_url)`:**

Follow RESEARCH.md skeleton exactly. Key points:

- Startup: `session = BrowserSession()`, `page = await session.start()`, `frame = await navigate_to_activity(page, channel_url)`, `await preflight_check(page)`, create `TilePlacer(page)`
- Store session: `self._session = session`
- Set state phase to RUNNING after successful startup
- Main while loop checks `not self._stop_event.is_set()`

- Step 1: `turn_state = await poll_turn(page)` — if game_over, post `build_gameover_embed` and break
- Step 2: Vision with retry — wrap `capture_canvas + extract_board_state` in try/except. On failure, retry once. On second failure, log and `continue` (skip cycle, re-enter poll_turn). CRITICAL: use `ch_state.mode` for extract_board_state.
- Step 3: Move generation — `moves = await asyncio.to_thread(find_all_moves, board, rack, self.bot.gaddag, ch_state.mode)`. MUST use asyncio.to_thread (CPU-bound, see RESEARCH.md pitfall 6). If moves exist, `selected = await asyncio.to_thread(self.bot.difficulty_engine.select_move, moves, ch_state.difficulty)`, then `candidates = [selected]` (pass single-item list so caller knows what was played — see RESEARCH.md open question 1). If no moves, `candidates = []`.
- Step 4: Increment turn_count and state.turn_count. Call `accepted = await placer.place_move(candidates, rack)`.
- Step 5: Post status — if accepted and candidates, `channel.send(embed=build_turn_embed(candidates[0], state.turn_count))`. Else `channel.send(embed=build_swap_embed(state.turn_count))`.
- After posting, check stop event again before next iteration.

- Exception handling:
  - Outer try wraps entire loop body (after startup)
  - Catch browser/navigation errors during the turn cycle and attempt reconnection via `_attempt_reconnect`
  - `except asyncio.CancelledError: raise` (always re-raise)
  - `except Exception`: log, post error embed to channel, break
  - `finally`: reset state to None, clear stop event, close session

**Reconnection `_attempt_reconnect(self, page, channel_url)`:**

- RECONNECT_DELAYS = [5, 15, 30]
- For each delay: try `navigate_to_activity(page, channel_url)`, return frame on success, sleep delay on failure
- If all 3 fail, raise RuntimeError

**Browser crash recovery:**

Inside the main loop, if an exception looks like a browser crash (catch `Exception` broadly, check if it's from patchright internals — `TargetClosedError` or similar), try:
1. Close old session
2. Create new BrowserSession, start(), navigate_to_activity()
3. Create new TilePlacer with new page
4. Continue loop

If browser relaunch also fails, fall through to the outer except which posts error and stops.

**`cog_unload(self)`:**
- If `self._loop_task and not self._loop_task.done(): self._loop_task.cancel()`
